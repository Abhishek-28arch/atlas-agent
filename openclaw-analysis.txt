  WHAT IS OPENCLAW?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

OpenClaw (formerly Clawdbot / Moltbot) is an open-source personal AI assistant
that runs on YOUR machine and is controlled via any chat app you already use —
WhatsApp, Telegram, Discord, Slack, Signal, or iMessage.

The core idea: instead of opening a browser tab to chat with AI, you message it
like you would a person. It lives on your computer 24/7, has persistent memory,
can control your entire system, browse the web, write code, manage files, and
extend itself with new skills — all through a simple chat message.

Built by: Peter Steinberger (@steipete) and the open-source community
GitHub: https://github.com/openclaw/openclaw
Docs: https://docs.openclaw.ai
Skills marketplace: https://clawhub.ai

Sponsors: OpenAI, Vercel, Blacksmith, Convex

  THE 6 CORE PILLARS (What Makes It Work)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. RUNS ON YOUR MACHINE
   ─────────────────────
   - Works on macOS, Windows, AND Linux
   - Supports Anthropic (Claude), OpenAI (GPT), AND local models
   - Private by default — your data never leaves your computer
   - Can also be self-hosted on a server (Raspberry Pi, VPS, etc.)
   - Users have run it on: Mac Mini, MacBook, Raspberry Pi, Hetzner VPS
   - No mandatory cloud dependency

   HOW TO BUILD THIS FOR JARVIS:
   → Use Ollama to run Qwen locally on Pop!_OS
   → Everything stays on your machine
   → No API costs

2. ANY CHAT APP (Channels / Interfaces)
   ──────────────────────────────────────
   OpenClaw connects to these messaging platforms as "channels":
   - WhatsApp (via unofficial bridge — risk of ban)
   - Telegram (official bot API — safest)
   - Discord (official bot API)
   - Slack (official API)
   - Signal (via signal-cli)
   - iMessage (macOS only)
   - Web dashboard (browser-based control UI at localhost:18789)
   - macOS menubar companion app (Beta)

   You message it like a coworker in DMs or group chats.
   It responds, takes actions, and reports back — all inside your chat app.

   HOW TO BUILD THIS FOR JARVIS:
   → Start with Telegram (easiest, most reliable)
   → Add Discord second
   → Build a local web dashboard (FastAPI + React)
   → Skip WhatsApp (ban risk) or add it later

3. PERSISTENT MEMORY
   ──────────────────
   - Remembers you across sessions permanently
   - Builds a "second brain" as you chat with it
   - Context and memory live on YOUR computer, not a server
   - Memory persists 24/7 even when you're not talking to it
   - Remembers preferences, past conversations, personal details
   - Memory is searchable and can be referenced across agents
   - Works with tools like Obsidian for note-taking integration

   HOW TO BUILD THIS FOR JARVIS:
   → ChromaDB for vector-based semantic memory (RAG)
   → SQLite for conversation history
   → Watchdog to index your files automatically
   → Store memory locally in ~/.jarvis/memory/

4. BROWSER CONTROL
   ─────────────────
   - Can browse any website
   - Fill out forms automatically
   - Extract data from any page
   - Log into websites on your behalf
   - One user had it open Google Cloud Console and configure OAuth by itself
   - Can check flight prices, book appointments, submit reimbursements
   - Works headlessly (invisible browser) or visibly

   HOW TO BUILD THIS FOR JARVIS:
   → Playwright (Python) for browser automation
   → Headless Chromium for invisible browsing
   → Add screenshot capability for vision feedback

5. FULL SYSTEM ACCESS
   ────────────────────
   - Read and write any file on your computer
   - Run terminal/shell commands
   - Execute scripts (Python, Bash, etc.)
   - Can be sandboxed (safe mode) or given full access
   - Users have had it: shut down their PC, turn off IoT devices,
     control smart home gadgets, manage processes
   - Runs Claude Code / Codex loops autonomously
   - Can fix failing tests, open PRs, deploy code — all autonomously

   HOW TO BUILD THIS FOR JARVIS:
   → subprocess module for terminal commands
   → PyAutoGUI for mouse/keyboard control
   → Safety layer: blacklist critical paths, require approval
   → Dry-run mode for testing

6. SKILLS & PLUGINS SYSTEM
   ─────────────────────────
   This is the KILLER FEATURE of OpenClaw.

   Skills are like plugins — small modules that teach the AI new abilities.
   - Community builds and shares skills on ClawHub (clawhub.ai)
   - You can build your own skills just by describing what you want
   - The AI can BUILD ITS OWN SKILLS — it writes and installs new skills
     by itself based on what it needs to do
   - Skills are hot-reloaded — no restart needed
   - Skills can be triggered by: user messages, cron jobs (scheduled),
     webhooks from external services, or heartbeat intervals
   - One user asked it to "create a skill for Todoist" — it wrote and
     installed the skill entirely on its own within Telegram chat

   EXISTING COMMUNITY SKILLS (examples from user testimonials):
   - Gmail control
   - Google Calendar management
   - GitHub automation
   - Spotify control
   - Philips Hue smart lights
   - Obsidian notes integration
   - Twitter/X posting
   - WordPress management
   - Todoist task management
   - WHOOP fitness tracker
   - Sentry error monitoring
   - Flight search CLI
   - Hetzner server management
   - Air purifier control (IoT)
   - VirusTotal security scanning (official partnership)
   - Custom meditation generator with TTS + ambient audio
   - RSS feed reader
   - Video generation (Sora2) with watermark removal

   HOW TO BUILD THIS FOR JARVIS:
   → Create a skills/ folder with Python modules
   → Each skill = one Python file with a standard interface
   → Jarvis loads skills at startup and hot-reloads on changes
   → Skills can be triggered by: chat message, cron, webhook
   → Let Jarvis write new skills when asked

FEATURE 1: GATEWAY SYSTEM
──────────────────────────
Description:
  The "Gateway" is the central server that runs on your machine. It is the
  brain that connects all channels (Telegram, Discord, etc.) to the AI model.
  All messages flow through the Gateway. It runs as a background daemon/service
  so it's always on even if you close the terminal.

Technical details:
  - Runs on port 18789 by default
  - Built with Node.js
  - Has a web Control UI at http://127.0.0.1:18789/
  - Can be started as a system service (auto-starts on boot)
  - Supports environment variables for custom config paths
  - Commands: openclaw gateway start/stop/status

How to build:
  → FastAPI server running in background (Python)
  → systemd service on Pop!_OS to auto-start
  → WebSocket for real-time updates
  → REST API for skill integration

FEATURE 2: PERSONA / ONBOARDING SYSTEM
────────────────────────────────────────
Description:
  When you first install OpenClaw, it runs an "onboarding wizard" that:
  - Asks about you (name, preferences, goals, timezone, work style)
  - Sets up the AI's personality and name (users name it Claudia, Jarvis,
    Brosef, Ema, Shelly, etc.)
  - Configures which channels to connect
  - Configures which AI model to use
  - Creates a system prompt specific to YOU

  This persona is persistent — the AI knows who you are and behaves
  consistently as "your" assistant across all sessions.

How to build:
  → First-run CLI wizard asking key questions about the user
  → Save answers to config.yaml
  → Generate a custom system prompt from the answers
  → Let user name their assistant

FEATURE 3: HEARTBEAT SYSTEM (Proactive AI)
───────────────────────────────────────────
Description:
  This is a UNIQUE feature that most AI assistants don't have.
  OpenClaw proactively reaches out to YOU — without being asked.

  Heartbeats are scheduled background tasks that run automatically:
  - Morning briefing: daily summary of tasks, weather, calendar, news
  - Reminders: "Leave now, traffic is bad on your route to pickleball"
  - Progress updates: "Running tests... 5 iterations done, still working"
  - Proactive suggestions: "You have 3 unanswered emails from last week"
  - Background monitoring: watches for events and notifies you

  Users describe this as the feature that makes it feel "alive."
  One user: "Apparently it checks in during heartbeats!? A kinda awesome surprise!"

How to build:
  → Python cron scheduler (APScheduler library)
  → Each heartbeat = a scheduled function that runs and sends Telegram message
  → Examples:
     - Every morning at 8am: send daily briefing
     - Every hour: check for urgent emails
     - Every 5 minutes during a task: send progress update
     - Every evening: summarize what was accomplished today

FEATURE 4: MULTI-AGENT SUPPORT
────────────────────────────────
Description:
  OpenClaw supports running multiple instances (agents) simultaneously.
  - One user cloned their assistant and ran 3 instances concurrently
  - Agents can communicate with each other
  - Can spawn sub-agents for parallel tasks (e.g., Claude Code, Codex)
  - Memory can be shared across agents
  - Used for: running tests while answering messages, parallel research

How to build:
  → Python multiprocessing or threading
  → Each agent = separate process with shared memory DB
  → Message queue between agents (Redis or simple JSON queue)
  → Start simple: one agent first, add multi-agent later

FEATURE 5: SELF-MODIFICATION / SELF-IMPROVEMENT
─────────────────────────────────────────────────
Description:
  This is the most mind-blowing feature of OpenClaw.
  The AI can modify itself — write new skills, edit its own system prompt,
  install new tools — all without you doing anything.

  Real examples from users:
  - "It combined tools in unexpected ways and added skills and made edits to
    its own prompt that were hot-reloaded"
  - "Claw can just keep building upon itself just by talking to it in Discord"
  - "Asked it to build a skill for Todoist — it did and started using it"
  - One user asked for flight search — it built a terminal CLI tool itself
  - Another asked for WHOOP data — it figured out the API and built the skill
  - It opened Google Cloud Console, configured OAuth, got an API key — itself

  How it works:
  - AI writes a new Python/JS skill file
  - Saves it to the skills/ folder
  - System hot-reloads the new skill automatically
  - AI immediately has the new capability

How to build:
  → Give Jarvis write access to skills/ folder
  → Create a "write_skill" tool that saves Python files
  → Use watchdog to hot-reload new skill files
  → Let Jarvis use Claude Code / subprocess to write and test skills
  → Add a skill approval gate (you confirm before loading new skills)

FEATURE 6: WEBHOOK / EVENT TRIGGERS
─────────────────────────────────────
Description:
  OpenClaw can receive webhooks from external services and react to them.
  This means external events trigger AI actions automatically.

  Example: Sentry (error monitoring) sends a webhook when your app crashes.
  OpenClaw receives it, reads the error, fixes the code, opens a PR — all
  automatically without you doing anything.

  Other webhook uses:
  - GitHub PR opened → AI reviews the code
  - Payment received → AI sends thank you email
  - Server CPU spike → AI investigates and fixes
  - New form submission → AI processes and responds

How to build:
  → FastAPI webhook endpoint (POST /webhook)
  → Register the URL with external services
  → Map webhook events to skill triggers
  → Queue incoming webhooks for AI processing

FEATURE 7: VIRUSTOTAL INTEGRATION (Security)
─────────────────────────────────────────────
Description:
  OpenClaw officially partnered with VirusTotal to scan skills for malware.
  Before installing a community skill from ClawHub, it's scanned for
  malicious code. This ensures the skills marketplace is safe.

How to build:
  → When adding community skills, run them through a safety checker
  → Use bandit (Python security linter) to scan skill files
  → Require review before installing third-party skills

FEATURE 8: MODEL FLEXIBILITY
──────────────────────────────
Description:
  OpenClaw is model-agnostic — you choose which AI model powers it:
  - Anthropic Claude (requires API key / Claude Max subscription)
  - OpenAI GPT models
  - Local models (mentioned: MiniMax M2.5, local Ollama models)
  - Can proxy through other subscriptions (one user routed CoPilot as API)
  - Users can switch models anytime

  One user used their Claude Max subscription through OpenClaw and hit
  rate limits, then switched to CoPilot via a proxy — all configured
  through conversation.

How to build:
  → config.yaml model selection
  → Ollama for local models (Qwen 8B / 3B)
  → Support OpenAI-compatible API format
  → Model auto-switching based on task complexity

FEATURE 10: PAIRING / DM SAFETY
─────────────────────────────────
Description:
  When you first connect a channel (e.g., Telegram), OpenClaw uses a
  "pairing" system to verify it's really you. This prevents strangers
  from sending commands to your assistant.

  - Send a pairing code from your chat app
  - Only paired users can control the assistant
  - Additional approval prompts for sensitive actions

How to build:
  → Store authorized Telegram user IDs in config
  → Require /start pairing command on first use
  → One-time verification code
  → Whitelist-only approach (only YOUR ID works)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Each skill = one Python file with this standard structure:

  # skills/example_skill.py

  SKILL_NAME = "example"
  SKILL_DESCRIPTION = "What this skill does"
  SKILL_TRIGGERS = ["keyword1", "keyword2"]  # words that activate this skill

  def run(input: str, context: dict) -> str:
      # Do the thing
      result = do_something(input)
      return f"Done: {result}"

  def schedule() -> dict:
      # Optional: run on a schedule
      return {"interval": "1h", "action": "check_emails"}

The skills loader:
  - Scans skills/ folder at startup
  - Imports each skill file
  - Registers skill triggers with the brain
  - When a message matches a trigger, runs the skill
  - Watchdog detects new/modified skill files → hot-reload
